oTree

Background

  • oTree started in 2013 at ETH Zurich, Daniel Chen group
  • Funded by Daniel Chens 5-year ERC grant
  • Homage to Fischbachers z-Tree
  • Published paper: http://www.otree.org/oTree.pdf
    • Citation required

Kinds of studies

  • Controlled behavioral experiments in economics, psychology, and related fields
  • Multiplayer strategy games, like the prisoner's dilemma, public goods game, and auctions
  • Surveys and quizzes

Concept

  • Code your experiment
    • Using Python and Django
  • Put it on a server
    • Your university's or hire your own
  • Share the link with participants
  • Watch the data come in (literally)

Features

Features

3 key features

Feature #1: Interaction

  • oTree enables true interaction between participants
  • Examples: trust game, prisoner's dilemma

Feature #2: Flexible

  • oTree is open source, therefore completely under your control
  • But is also easily expandable without touching core code, for example using Javascript libraries
  • Examples: mouse tracking, venn task, webcam support

Feature #3: Online

  • oTree enables complex online studies
  • Part of the solution to large sample sizes is to run studies online
  • Amazon Mechanical Turk integration
  • Example: Bastian's presentation, personality and trust study

Limitations

  • Dropout
  • Screen sizes
  • Requires programming

Required skills

  • Setting up a server: let your IT people do this
  • Python: Learn Python the Hard Way, oTree Python Tutorial
  • Django: oTree tutorials

  • oTree: oTree documentation, Mailing list

  • HTML, CSS, Javascript: any tutorial, Stack Overflow

Conclusion

oTree is a very well-designed platform that can be used to address (at least) three key challenges for social scientists:

  • true interaction between participants
  • flexible experimental design
  • run online studies to acquire large sample sizes
